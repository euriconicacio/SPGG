Input file template


For a proper working of SPGG, it is necessary that the input data are correctly arranged in the input file, following the input file template. It is only accepted ".xls" input file formats with the following formatting:


    • Column #1 - Id

Id number. It is suggested the usage of sequential numbering started in 1 (e.g.: 1, 2, 3, ...) or any numerical codes as desired by the user.


    • Column #2 - Points latitude in decimal degrees

It is necessary to include the signal and as much decimal places as possible in order to determining results with equivalent precision (e.g.: -25.455528833)


    • Column #3 - Points longitude in decimal degrees

It is necessary to include the signal and as much decimal places as possible in order to determining results with equivalent precision (e.g.: -49.237430388)


    • Column #4 - Elipsoidal height or "Height over ellipsoid", in meters.

This field is not necessary for "geoid" functional, being requested only for all the other functionals, in meters (e.g.: 911.134).


WARNING! IMPORTANT! - PRECAUTIONS WITH THE INPUT FILE FORMAT


    1. Do not insert a header line! Type directly the points information, as follows in figure 16:


Figura 16 - Exemplo de arquivo de entrada.


    1. Do not use non-numeric characters to fill in the points Id in the first column;
    2. Do not leave blank lines/spaces between the filled lines;
    3. Use dot (".") as a decimal places separator, do not use comma (",");
    4. Use just one spreadsheet file to as much points as you need.


A input file template can be downloaded in the following link: Input File Tempalte.
