Options and features


In this page, it is carried out an superficial approach on SPGG buttons and features. That because it is presumed that the user has already had previous contact with ICGEM web service for gravity field functionals calculation. In case of not possessing such knowledge, it is recommended a previous access to the  ICGEM website for familizarization.


First of all, it is shown a general vision of SPGG main window:


Figure 3 - SPGG main window.


We will now walk through each button, option and feature. Initially, it is noticed that there are two tabs in the application, as follows::


  • "Extract Generation" Tab - initial tab that should be used for system operation; a visualization of this tab is shown in figure 3.


  • "About" Tab - tab with authorship information and with the indication of the address to access this guide; a visualization of this tab is shown in figure 4.


Figure 4 - SPGG "About" tab.



Keeping the focus on the main tab, we can find the following groups of features:


  • "Model and functional parameters" group -


This group containts the specifications of the model a user wants to work with.


    • Directory - the user may choose which type of model he wants to work with among the options "Longtime Model", "Model from Series", "Topography related Model", "Celestial Object Model" and "Topography";


    • Model - the user mau select which model he wants to use to generate the geopotential functional. Such choice must be performed after choosing among the directory options. Exampples of models are: GGM05C, EIGEN-6S4, EGM2008, and others. For more information on the available models, visit ICGEM Table of Models.


If the user wants to generate a "Model from series" or a "Celestial Object Model" and chose such options in the "Directory" field, other options will be displayed in the "Model" field. For example, by choosing  "Model from series", this field will be filled in with groups of time series models, such as "GRACE_monthly/JPL Release 05" and "GRACE_daily/ITSG-Grace2014". In the other hand, by choosing "Celestial Object Model", this field will be filled in with options of other celestial objects available for modelling, such as "Mars", "Venus" and "Moon (of the Earth)". In both the previous cases, new options will be displayed at the "Other options" field.


Please, notice that in the first option (choosing "Model from series"), a little loading time - not longer than a few seconds - will be necessary in order to SPGG obtain all of the available options straightly from ICGEM website.


    • Other options - Thsi field is unabled by default, being enabled only if the user chooses "Model from series" or "Celestial Object Model" in the "Directory" field and chooses further specification parameters in the "Model" field.


For example, if the user wants to generate a Mars geopotential functional extract, he mus select "Celestial Object Model" in the "Directory" field, "Mars" in the "Model" field, and specify which model he wants to use in the "Other options" field. A similar procedure must be adopted in case of "Model from series" selection in the "Directory" field.


    • Functional - In this field, the user must select which geopotential functional he wants to use. The available functionals vary according to his previous options. Examples of functionals are: height_anomaly, geoid, gravity_disturbance, and others. For further information on the functionals, please refer to the ICGEM documentation [Definition of Functionals of the Geopotential and Their Calculation from Spherical Harmonic Models (STR09/02)].


    • Reference System - In this field, the user must select which reference system he wants to use in order to generate his results. The selection options vary according to the previous choices made by the user. General standard options for Earth modelling are WGS84, GRS80, GRIM5, EGM2008, GRS67, JGM3, WGS72 and NIMA96. As the user selects his desired options information on the reference system defining parameters are displayed at the following fields ("Radius", "GM", "Flat" and "Omega").


An update provided in this new version of SPGG is the possibility of choosing a User Defined Reference System: an user can provide the following parameters in order to calculate the functionals according to other reference system differente from the standard ones. In order to perform such action, one must select the "- user defined -" option in the "Reference System" field and, after that, provide the information on the following fields. These fields are initially unabled for filling and, in case of selection of standard systems, automatically filled with standard values; they are only enabled for filling after the "- user defined -" selection.


      • Radius - the reference radius (e.g., for GRS80, the radius is equal to 6378137.0);
      • GM - the product of gravitational constant and mass of the Earth (e.g., for GRS80, GM is equal to 3.986005e+14);
      • Flat - the flattening of the reference ellipsoid (e.g., for GRS80, the flattening factor is equal to 298.257222101);
      • Omega - the angular velocity of the Earth  (e.g., for GRS80, Omega is equal to 7.292115e-5).


    • Tide system - In this field, the user must select which tide system he wants to use. The available options are use unmodified model (by default), tide free, mean tide and zero tide, and may vary according to the previous choices.


    • Zero Degree Term - In this field, the user must choose if he wants or not to use the zero degree term in the spherical harmonic development of the functional. The available options are: yes (by default) and no.




  • "Points selection" group -


    • "Open" button -

The user must select which ".xls" file contains the list of points properly formatted por th application usage. By clicking this button, it is displayed a dialog box as the presented in figure 5.        


Figure 5 - Dialog box for ".xls" input file selection.


In case of selecting a file format different from the accepted, it is displayed a message as shown in figure 6.


Figure 6 - Error message for invalid input file format.


    • Points selection label - Exactly below the "Open" button, there is a label informing the current status of the point selection. Initially, it displays a "No points loaded." status. As soon as the user chooses a working spreadsheet file, its status changes do "OK! File selected!".


WARNING! IMPORTANT! - POINTS SELECTION LABEL STATUS CHANGE


Please, notice that the "Points selection" label change is only related to correctly loading a ".xls" input file; it does not inform that this input file is correctly formatted! To check correction in formatting, please refer to the "Input file template" section of this guide.



  • "Truncation" group -


    • "Gentle cut" selection - In this field, the user must inform from which degree of the model spherical harmonic development he wants to start a gentle cut; this value must be inferior thant de "Max degree" value; by default, it is presented the value 9999, meaning that no gentle cut will be applied at all.


    • "Max degree" selection - In this field, the user must inform up to which degree of development he wants to obtain his extract; by default, it is presented the value 9999, meaning that the model will be developed up to its maximum possible degree (e.g., EGM2008 model enables development up to degree 2190).


               




  • "Calculation" group -


    • "Test Connection" button -

This button should be pressed by the user in order to test his internet connectivity.


    • "Clear" button -

This button clears all the options and fields already filled in.        


    • "Generate extracts" button -

       This button starts the calculation of the GGM functionals extracts on the selected points. It must be pressed only after the correct fulfillment of the other fields.



Obs.: Differently from the previous version, SPGG v2.0 does not have a "Update options" button, in order to update the available models provided by ICGEM. That is becaus in this version the options loading is performed in real-time procedure: in the exact moment an user clicks the "Directory", "Model", "Other options" and "Functional" fields, the application loads all the available at the ICGEM Calculation Service.



  • "Processing" group -


    • "Estimated Time" label - This field displays the estimated time up to finishing the calculation. It is based on a maximum estimative of 20 seconds per calculation point.


    • "Elapsed Time" label - This field displays the real elapsed time since the beginning of the calculation (when the user pressed the "Generate extracts" button; it is updated after each point is finished.

       

    • "Progress" bar - This field provides a percentual idea of the calculation activities progress, varying from 0 up to 100%, as new points are calculated. Complementarly to the progress bar and displayed below it, there is a field that shows the current point under calculation in a comparative way to the total number of points. For instance, if your points spreadsheet contains 100 points and 37 of them were already calculate, this label will display the status "37 of 100"; this field is updated after each point is finished.




  • "Results" group -


    • "Open results" button -

               After all the calculations are finished, it is possible opening the results in the output file by pressing this button. Please, notice that this option is only valid AFTER THE EXTRACTS GENERATION! In case of pressing this button previously to performing the calculation, an error message is displayed as shown in figure 7.


Figure 7 - Error message for output file opening.


    • "Close" button -

               This button closes the application.
