SPGG Operation


In this section, it is provided an example of SPGG operation, since the input file formatting up to the data extraction of the output file. For a general approach, check the video displayed at this link.


Consider the following ten steps ahead, with the objective of determining the geoid height of 3 known points, according to a pre-established model and its specifications.


    1. Data arrangement


Suppose that, as stated above, we wanto to determine de geoid height of 3 points. Our first task will be arranging their known positional data in any application, preparing to generate our input file. In this moment, we must perform this task according to the order and the number of columns that will be used by SPGG. In case of pre-arranged data, skip this step.

Figure 8 contains an examplo of data arrangement.


Figure 8 - Data arrangement.



    1. Input file formatting


Our next step is to format our data according to the stated in the "Input file template" section of this guide. The result is show in figure 9.


Figure 9 - Formatted input file.


Remember that it is necessary to save the input file in ".xls" format!



    1. Starting SPGG


Next, assuming that SPGG is already correctly installed, we should initialize it. Figure 10 contains a summary of its call and initial screen.


Figura 10 - SPGG call and initial screen.



    1. Parameters and truncation setting and points selection


Afterwise, we fill in all of the fields in SPGG, as stated in "Options and Features" section, in accordance with figure 11.


Figure 11 - SPGG fields filled with wanted information.



    1. Connection test


After reaching this point, it is recommended that the user tests his internet connection. Such test is performed by clicking the "Test Connection" button, and its result is displayed in figure 12.


Figure 12 - Positive result for internet connectivity test.



    1. Calculation beginning


In the sequence, the user must presse the "Generate extracts" button in order to, in fact, start the calculation.



    1. During the calculation


During the calculations, your screen may even get the "No answer" aspect, depending on you operation system. However, the calculation is being held, as it can be stated by the "Processing" group status in the application - show in figure 13.


Figure 13 - Application during processing.



    1. Calculation end


In the end of the calculation, a success alert is displayed, as shown in figure 14.


Figure 14 - Calculation finished alert.



    1. Opening the output file


The output file opening can be performed by pressing the "Open results" button, or searching it directly in the folder indicated in the "Output file pattern" section of this guide. A visualization of its opening is presented in figure 15.


Figura 15 - Opening the output file.



    1. Interpretation of output file


An interpretation of the output file should be based on the "Output file pattern" section in this guide.
