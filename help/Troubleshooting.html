Troubleshooting


Collection of reported and solved issues


The following are some issues that have been reported by users and their solutions.


    • Issue #1: Interrupted connection; lost of calculated points up to the interruption moment.


      • Description: during the calculation of several points, your internet connection is interrupted;


      • Solution or recommendation: unfortunatelly, the lack of internet connectivity during the application execution makes it fail and close. However, the calculated points up to that moment are not lost. In the folder where the spgg2.tar.gz file was uncompressed, it is created a text file named temp.txt. This file contains a sort of log file of the calculated points, as displayed in figure 18. When the internet connection gets restablished, you can re-format your input file by excluding the already calculated points.


Figura 18 - Log file of the calculated points.


    • Issue #2: Issues related to xvfb installation.


      • Description: you do not have one of the packages necessary to run the application;


      • Solution or recommendation: it is necessary to install the missing package and you probably will need to check issue #3; follow these steps:


        1. Install xvfb latest version: sudo apt-get install xvfb.


    • Issue #3: Issues related to Chromium installation on Ubuntu.


      • Description: you have both Linux and Chromium installed in you PC; however, as you try starting SPGG, it informs that your internet connectivity is working, but the application suddenly closes during operation due to Chromium driver issues;


      • Solution or recommendation: in some situations, Chromium web browser installation is not followed by proper installation of ChromeDriver, which is also necessary for this application execution. It is recommended the access to this link to ensure the correct installation of both. If you prefer, follow this step-by-step in Linux command line:


        1. Check if you have the latest version of Chromium web browser installed:  chromium-browser -version
        2. In case negative, install Chromium latest version: sudo apt-get install chromium-browser
        3. Download the ChromeDriver version more appropriate to your OS here.
        4. Uncompress the chromedriver.zip file.
        5. Move the uncompressed file to the /usr/bin folder with the command sudo mv chromedriver /usr/bin
        6. Go to the /usr/bin folder and change the file permissions with a command equivalent to chmod a+x chromedriver
        7. Save your work in other applications, restart you system em run SPGG again.


    • Issue #4: Issues related to incompatibility between Google Chrome and Chromium Web Browser on Ubuntu.


      • Description: you have Linux, Chromium and Google Chrome (from other Linux distributions) installed in your PC; when you try to run SPGG, you face an error that ends similarly to the following: http.client.RemoteDisconnected: Remote end closed connection without response


      • Solution or recommendation: this issue happens due to an incompatibility between Google Chrome itself of other Linux distributions and the Chromium Browser. You must uninstall the existing versions of Google Chrome, as follows:


        1. Uninstall Chrome stable version, if there is one: sudo apt-get remove google-chrome-stable
        2. Uninstall Chrome unstable version, if there is one: sudo apt-get remove google-chrome-unstable
        3. Uninstall Chrome beta version, if there is one: sudo apt-get remove google-chrome-beta
        4. Remove spare packages automatically installed, as followed: sudo apt-get autoremovef (be careful performing this action if working with Debian-based distributions of Linux);
        5. Update the packages list, as follows: sudo apt-get update
        6. Run SPGG again.


      • Observation: special acknowlegment to Mr. José Carrión by his support on identifying and reporting this issue.
